Settings in mUzima

The mUzima program settings can be accessed through the home page. Click on the hamburger icon on the top right corner.

OpenMRS Settings

This setting specifies the url of the server used by mUzima to access data. A mUzima user can change this address by clicking on the address field. Username and Password are changed only on the server.

mUzima Settings

These are application level settings.

  • Session timeout if the program is not in use.This is a security aspect of mUzima designed to protect patient information.
  • The automatic form save interval (minutes) is used to save the form without healthcare provider intervention.
  • A real-time sync that allows you to automatically upload completed forms to the server.
  • Require medical record number will make medical record number to be required if it is enabled.
  • Duplicate form data warning if form has been filled more than once; alerts the user when saving data.
  • Font size; the user can choose the font size between small, medium and large.
  • Enable Day mode is used to toggle between dark mode(default) and light mode
  • Enable SHR will enable/disable the Shared Health Records(Smart card feature) on the mobile application
  • Enable GPS will enable/disable the collection of GPS information as well as the the geo-mapping feature
Provider/Location Settings

These are provider/location level settings

  • Default encounter provider : the user logged will automatically be pre-filled as the patient's service provider while filling in a form.
  • Manage providers : used to get other service providers from the Server or remove them from the mUzima App.
  • Default Encounter Location : the encounter location will be used as a default encounter location during an encounter and pre-filled in a form.
  • Manage Locations : the user can get encounter locations from the server or remove an encounter location in mUzima.
  • Manage Concept : This setting is used to get more concepts from the server or remove the concepts from the device.
